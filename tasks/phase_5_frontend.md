# Phase 5: Frontend UI Development

**Goal:** Build the user-facing Client View and the Admin Dashboard.

**Tasks:**
1. **Client View:** Build the UI to input travel preferences and display approved itineraries.
2. **Admin Dashboard:** Build the HITL interface to fetch, display, and approve/reject pending itineraries.
3. **UX & Latency Handling:** Implement async polling (or WebSockets) to provide live status updates to the user (e.g., "Researching flights...", "Evaluating options...") during the agent workflow.
4. Apply premium styling using Vanilla CSS.

**Definition of Done:** A complete, interactive web interface connected to the FastAPI backend.
